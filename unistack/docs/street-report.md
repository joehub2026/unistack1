# Street Report — UniStack: INES Digital Notice Board + Marketplace

## 👤 User Quote
> "Every time I post something for sale on the WhatsApp group, it gets buried in 200 messages within an hour. I once lost a buyer for my laptop because they couldn't find my post again. And someone in my class got scammed buying a phone from someone nobody knew."
— **3rd Year CS Student, INES-Ruhengeri, 2025**

---

## 🔍 Exact Problem Being Solved
Students at INES-Ruhengeri currently use informal WhatsApp groups to:
- Buy and sell second-hand items (phones, books, laptops, clothes)
- Share housing/accommodation information
- Post announcements and academic notices

This creates 3 critical problems:
1. **Posts get lost** — high message volume buries listings within hours
2. **Scams happen** — no identity verification or moderation layer
3. **No categorization** — housing, sales, and announcements are mixed together

---

## 🎯 Chosen Scenario
**UniStack — INES Digital Notice Board + Marketplace (Advanced)**  
Project C from the "From Street to Stack" assignment.

---

## 📦 Scope for This Assignment
A student-only web platform with:
- Role-based login (Student / Moderator / Admin) using school email pattern validation
- Post types: For Sale | Housing | Announcement
- Approval workflow: Pending → Approved → Rejected
- Student dashboard with personal post tracking
- Flagging/report system for suspicious posts
- Search and category filtering
- Real-time-like JS polling simulation (10s refresh)
- Admin user management panel
- Moderator approval interface

**Out of scope (future versions):**
- Real-time chat
- Payment integration
- Mobile app
